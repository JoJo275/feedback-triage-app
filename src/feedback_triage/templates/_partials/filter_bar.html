{# Filter bar -- inbox / feedback list. Pure markup; state is managed client-side by `inbox.js`, which reads the URL query string on load and writes back via `history.replaceState`. The available filters mirror the v2 list endpoint's query parameters (`docs/project/spec/v2/api.md` — `GET /api/v1/feedback`). The page-mode flag controls the default status filter: * inbox -> { new, needs_info, reviewing } * feedback -> all statuses The user can override the inbox default via the dropdown. #}
